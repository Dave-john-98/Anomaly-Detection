Creating Feature 9 - Deviation of Average Covered Charges

To improve our anomaly detection, we introduce a new feature that calculates how far each providers Average Covered Charges deviate from the mean for their corresponding DRG Definition.

This feature is more informative for PCA and KNN since it introduces variability at the row level, as opposed to having repeated values within each DRG Definition. It will help detect providers with charges that significantly differ from others in the same category.

Creating Feature 11 - DRG Complexity Score

To capture the complexity of the procedures handled by each provider, we create a DRG Complexity Score. We assign a complexity score to each DRG Definition based on its average total charges, assuming more costly procedures are more complex.

We refine the DRG Complexity Score to calculate a weighted average complexity score for each provider, based on the procedures they handle. The providers score is weighted by the number of cases of each DRG they handle.

This feature will give each provider a unique complexity score based on the mix of procedures they handle.

In[6]:
# Calculate the DRG complexity score based on Average Covered Charges for each DRG Definition
drg_complexity = inpatient_data.groupby('DRG Definition')['Average Covered Charges'].mean()

# Map the DRG complexity score to the dataset
inpatient_data['DRG Complexity Score'] = inpatient_data['DRG Definition'].map(drg_complexity)

# Calculate the weighted DRG complexity score for each provider
inpatient_data['Weighted DRG Complexity Score'] = (
    inpatient_data['DRG Complexity Score'] * inpatient_data['Total Discharges']

K-Nearest Neighbors (KNN) for Anomaly Detection

We now apply K-Nearest Neighbors (KNN) from the PyOD library to detect anomalies in the dataset. KNN works by calculating the distance between points and identifying those that are farthest away from their neighbors, which could indicate potential anomalies.

Steps:

  1. We will use the KNN implementation from PyOD.
  2. The contamination rate is set to 0.05, meaning we expect around 5% of the data to be anomalous.
  3. We will fit the KNN model on our training data and use the model to predict anomalies in both the training and test datasets.
In[9]:
# Import necessary modules
from pyod.models.knn import KNN
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# Split the dataset into training and test sets (70/30 split)
X_train, X_test = train_test_split(subset_data, test_size=0.30, random_state=42)

# Scale the training and test sets
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

# Train the KNN model
knn = KNN(contamination=0.05)  # Set contamination rate (expected outliers)
knn.fit(X_train_scaled)

# Get the KNN decision scores and predictions for the training data
y_train_scores = knn.decision_function(X_train_scaled)  # Decision scores for train set
y_train_pred = knn.predict(X_train_scaled)  # Predictions (0: normal, 1: anomaly)

# Get the KNN decision scores and predictions for the test data
y_test_scores = knn.decision_function(X_test_scaled)  # Decision scores for test set
y_test_pred = knn.predict(X_test_scaled)  # Predictions (0: normal, 1: anomaly)

# Define the utility function for counting anomaly stats
def count_stat(vector):
    # Because it is '0' and '1', we can run a count statistic.
    unique, counts = np.unique(vector, return_counts=True)
    return dict(zip(unique, counts))

# Display stats using count_stat for both training and test data
print(f"Anomalies (1) vs Normal (0) in Train Data: {count_stat(y_train_pred)}")
print(f"Anomalies (1) vs Normal (0) in Test Data: {count_stat(y_test_pred)}")

# Print the threshold used for defining anomalies (based on contamination rate)
print("The threshold for the defined contamination rate:", knn.threshold_)

Principal Component Analysis (PCA) is a dimensionality reduction technique that helps identify the most important components (directions) in the data that explain the maximum variance.
